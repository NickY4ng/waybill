# 大卡鹰眼优化方案 | 基于2026-03-05会议纪要

## 一、产品定位优化

### 当前问题
- "智能问数"给人感觉是"卖数据工具"
- 内部会觉得我们在卖数据，不是卖能力

### 优化方案
| 维度 | 优化前 | 优化后 |
|-----|-------|-------|
| **产品名称** | 智能问数 | 智能报表 / 分析报表 |
| **核心卖点** | 快速查询数据 | 行业深度分析能力 |
| **人设定位** | 数据查询工具 | 物流行业数据专家 |

### 关键话术
- ❌ "查询2025年6月北京到河北运单量"
- ✅ "老板让我分析A工厂和B工厂这个月的业务流，报告怎么展现？"

## 二、UI界面优化（三栏布局）

### 当前问题
- 对话窗口占主导，像豆包/文心一言
- 缺乏行业特性展示
- 没有体现数据能力和安全感

### 优化方案：三栏/四栏设计

```
┌─────────────────────────────────────────────────────────────┐
│  【左侧】会话列表    │    【中间】能力展示区      │    【右侧】工作日志    │
│                    │                            │                      │
│  - 历史会话         │  开屏："开启大卡鹰眼之旅"   │  ✓ 已加载河北省      │
│  - 新建会话按钮     │                            │    XX万条数据        │
│                    │  模板展示区（核心！）        │  ✓ 智能算法已启动    │
│                    │  ├─ 上下游分析模板          │  ✓ 隐私加密已开启    │
│                    │  ├─ 选址分析模板            │  ✓ 数据仅您可见      │
│                    │  ├─ 货物流向模板            │                      │
│                    │  └─ 环比对比模板            │  【情绪价值区】       │
│                    │                            │                      │
│                    │  对话区（下方/左侧）         │                      │
│                    │  （自然语言输入框）          │                      │
└─────────────────────────────────────────────────────────────┘
```

### 各栏设计要点

**左侧：会话管理**
- 历史会话列表（可折叠）
- 新建会话按钮
- 每个会话独立，避免话题混杂

**中间：能力展示（重中之重！）**
- 开屏要有吸引力，不能是空对话框
- 展示3-5个报告模板
- 突出物流行业特性
- 对话区放在次要位置

**右侧：工作日志（情绪价值）**
- 数据加载状态可视化
- 算法启动提示
- 隐私加密声明
- 数据安全提示
- **目的**：让用户感觉系统"在运转"、"专业"、"安全"

## 三、功能模块优化

### 模块划分

| 模块 | 名称 | 定位 | 交互方式 |
|-----|-----|------|---------|
| **模块1** | 智能报表（快速分析） | 简单查询、快速出结果 | 自然语言 → 一轮确认 → 出结果 |
| **模块2** | 深度分析 | 复杂报告、多轮交互 | 单独会话、专家角色、耗时较长 |

### 关键决策

1. **分开会话**
   - 快速分析和深度分析不在同一个对话框
   - 避免对话混乱，便于历史追溯
   - 通过"新建会话"或勾选进入深度模式

2. **弱化提数功能**
   - 不叫"原始数据导出"
   - 叫"加工数据集导出"
   - 作为辅助功能，非主要产品

3. **权限管理**
   - 系统提示词带入用户身份信息
   - 不同客户购买不同数据范围
   - 权限不够时引导购买升级

## 四、交互方式优化

### 当前问题
- 用户提问太明确："查询XX数据"
- 感觉像数据库查询工具

### 优化方案：场景化自然语言

**错误示例**：
> "查询2025年6月北京到河北的运单量"

**正确示例**：
> "下午我们部门老大要给CEO汇报，让我出一个报表体现我们工厂现在运营变好，我应该怎么去做？"

**正确示例**：
> "你帮我分析一下A工厂和B工厂这个月的业务流，数据报告应该怎么展现？"

### 核心逻辑
- 用户用自然语言描述业务场景
- 系统理解意图后反问确认
- 输出符合业务需求的分析报告
- 支持多轮调整（维度、样式等）
